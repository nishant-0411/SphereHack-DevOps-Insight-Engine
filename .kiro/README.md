# Kiro Workspace

This folder contains all planning, architecture, and documentation created with Kiro for systematic development.

## Structure

### `/specs/`
Feature specifications following the requirements → design → tasks workflow
- Each feature gets its own subfolder with `requirements.md`, `design.md`, and `tasks.md`

### `/planning/`
High-level planning and brainstorming documents
- Project roadmaps
- Feature ideas and exploration
- Technical decision records

### `/architecture/`
System architecture and design documents
- System diagrams
- API designs
- Database schemas
- Integration patterns

### `/prototypes/`
Quick experiments and proof-of-concepts
- Code sketches
- Technology evaluations
- Performance tests

### `/docs/`
Generated and maintained documentation
- API documentation
- User guides
- Development workflows
- Deployment guides

### `/flows/`
User flows, business processes, and workflow diagrams
- User journey maps
- Business process flows
- Development workflows

## Usage

1. **New Feature**: Start in `/specs/` with requirements gathering
2. **Architecture Changes**: Document in `/architecture/` 
3. **Quick Ideas**: Capture in `/planning/` or `/prototypes/`
4. **Documentation**: Maintain in `/docs/`

This structure supports both creative exploration and systematic development.
